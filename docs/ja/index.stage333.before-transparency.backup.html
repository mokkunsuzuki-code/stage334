REMEDA Stage328

QSP Evidence Match Gate

概要

Stage328は、Stage327で作った 再現証拠スキーマ を土台にして、 AIの脆弱性指摘と再現証拠が本当に一致しているかを判定します。

つまり、Stage327は 証拠を構造化する段階、 Stage328は その証拠がAI Claimと一致するか判定する段階 です。

判定モデル

AI Claim
↓
Reproduction Evidence
↓
same_target / evidence_files_present / sha256_bound / signature_present
↓
QSP Evidence Match Gate
↓
accept / pending / reject

判定ルール

accept：same_target / evidence_files_present / sha256_bound / signature_present がすべて true

pending：対象と証拠はあるが、SHA256または署名が不足

reject：対象が違う、または証拠ファイルがない

核心

Stage327は、再現証拠を 標準構造化 しました。

Stage328は、その構造化された証拠がAI Claimと一致するかを確認し、 accept / pending / reject に変換します。
